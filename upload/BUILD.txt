SPLIT DEPLOY — Frontend (aitrads.in) + Backend API (api.aitrads.in)
====================================================================

Frontend  → Hostinger File Manager → aitrads.in/public_html  (NO Docker)
Backend   → VPS Docker → https://api.aitrads.in  (NOT srv1831231.hstgr.cloud)

--- DNS (Hostinger) ---
  aitrads.in        → shared hosting (frontend files)
  api.aitrads.in    → A record → VPS public IP

--- VPS: purana Docker hatao (ek baar) ---
  cd /docker/crypto-ai-trads/crypto-ai-trads
  docker compose down -v --remove-orphans
  docker system prune -af --volumes

--- VPS: sirf backend deploy ---
  git pull
  cp .env.example .env
  cp backend/.env.example backend/.env    # keys edit karo
  docker compose up -d --build

  Test: curl http://api.aitrads.in/health
  Then:  curl https://api.aitrads.in/health

  SSL / ERR_SSL_PROTOCOL_ERROR fix (HTTP 308 but HTTPS fails):
  cd /docker/crypto-ai-trads/crypto-ai-trads
  git pull
  echo -e "API_HOST=api.aitrads.in\nACME_EMAIL=admin@aitrads.in" > .env
  docker compose down
  docker volume rm crypto-ai-trads_caddy_data crypto-ai-trads_caddy_config 2>/dev/null || true
  docker compose up -d --build
  sleep 20
  docker compose logs caddy --tail 40
  curl -sS http://api.aitrads.in/health
  curl -sS https://api.aitrads.in/health

  Logs mein "certificate obtained successfully" dikhna chahiye.
  Agar ACME fail ho: firewall me 80+443 open rakho, phir dubara volume wipe.

--- Browser ---
  App:  https://aitrads.in
  API:  https://api.aitrads.in/health

Firewall: ports 80 + 443 open. Port 8000 browser se mat kholo.

--- SAME VPS: 2+ Docker projects (IMPORTANT) ---
  Ek VPS par sirf EK container ports 80 + 443 bind kar sakta hai.
  Doosra project deploy karne ke baad SSL error = port fight / galat Caddy ne 443 pakda.

  Check kaun 80/443 use kar raha hai:
    docker ps --format "table {{.Names}}\t{{.Ports}}"
    ss -tlnp | grep -E ':80|:443'

  Caddy log "admin.json: no such file" = ERROR NAHI — naya ACME account ban raha hai (normal).
  Asli problem: "certificate obtained" ke baad bhi browser fail = doosra Caddy 443 par hai.

  FIX A — is project ka Caddy band, doosre project ke Caddy me api route add karo:
    cd /docker/crypto-ai-trads/crypto-ai-trads
    git pull
    docker compose down
    docker compose -f docker-compose.backend-only.yml up -d --build
    curl http://127.0.0.1:8000/health

    Doosre project ke Caddyfile me add (see caddy/snippet-api.aitrads.in.caddy):
      api.aitrads.in { reverse_proxy 127.0.0.1:8000 }
    phir doosre project ka caddy restart

  FIX B — sirf ye project 80+443 use kare (doosre project ka caddy ports hatao):
    docker compose down   # doosre project folder me
    cd /docker/crypto-ai-trads/crypto-ai-trads
    docker volume rm crypto-ai-trads_caddy_data crypto-ai-trads_caddy_config
    docker compose up -d --build
