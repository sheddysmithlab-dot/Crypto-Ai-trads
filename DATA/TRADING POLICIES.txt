RULE 3: The Entry Condition (Volume Surge 2x)
Volume Baseline: For every new candle on the selected timeframe, read the volume of the immediately preceding (previous) closed candle.

The 2x Trigger: Continuously monitor the live volume of the current forming candle.

Execution: The exact moment the live volume of the current candle becomes 2 times (2x) or greater than the total volume of the previous closed candle, you must immediately execute a BUY (Long) order.

Do not wait: Do not wait for the current candle to close. The trade must trigger the millisecond the 2x volume threshold is crossed.

RULE 4: The Dynamic Trailing Lock (Profit Protection)
(Merging our previous logic)

Activation: Once a Buy order is placed, monitor the profit. When the price reaches a profit margin equivalent to +0.07, activate the trailing lock.

Reversal Sell: If the price drops by exactly 0.02 from its highest recorded peak, execute a Market SELL order immediately to lock profits.

Hard Floor: Never execute the trailing sell at a profit lower than +0.05.

Halt System: Stop all further trading activity and send an "EMERGENCY STOP" alert to the Front-end.

RULE 6: True Net Profit & Fee Deduction Logic
1. Live Fee Calculation:

For every executed trade, instantly calculate the Entry Fee (Opening fee) based on Bybit's current Taker/Market fee rate applied to the total leveraged position size (e.g., $100).

Continuously calculate the Estimated Exit Fee (Closing fee) dynamically, based on the live current price of the asset.

2. Gross vs. Net Profit Tracking:

Do not trigger any trailing locks based on Gross Profit.

Always use this exact formula for decision-making:
True Net Profit = Gross Profit - (Entry Fee + Estimated Exit Fee)

3. Net Profit Activation (The 0.07+ Rule Updated):

The Trailing Lock must ONLY activate when the True Net Profit (after deducting all exchange fees) reaches +0.07.

If the Gross profit is +0.07, but the Net Profit is only +0.03 due to fees, the system must wait.

4. Net Profit Hard Floor:

The absolute minimum sell point (0.05 floor) must also be calculated as True Net Profit. The system must execute the sell order early enough to ensure that exactly +0.05 remains in the wallet after Bybit deducts the exit fee.

RULE 7: Execution Order Type (Instant Action Policy)
1. Mandatory Market Orders:

Entry: All Buy (Long) orders triggered by the "2x Volume Spike" condition MUST be executed strictly as Market Orders. The priority is 100% guaranteed entry the exact millisecond the volume condition is met.

Exit: All Sell orders (whether triggered by the Trailing Lock Reversal, the +0.05 Hard Floor, or the 2.5% Emergency Kill Switch) MUST be executed strictly as Market Orders to ensure instant exit without getting stuck in the order book.

2. Slippage & Taker Fee Sync:

Since the system will exclusively use Market Orders, the AI Agent must continuously fetch Bybit's current Taker Fee tier via API and apply it directly to the True Net Profit (Rule 6) calculations.

The system must accept minor slippage (price difference between trigger and execution) as a normal part of Market Orders, but immediately recalculate the trailing lock based on the actual filled price, not the requested price.

4. Action: CONTINUE (The 5% Escalation Logic):

How the Backend and Frontend Handle the 30 Seconds
To make this work flawlessly, the "30 seconds" logic actually needs to be managed by both the frontend and the backend so it can't be tricked.

The Backend Timer (The Source of Truth):
When the backend hits the 2.5% loss, it sends the alert to the frontend and starts its own internal 30-second stopwatch. If it doesn't receive a "CONTINUE" command from the frontend before the 30 seconds run out, the backend executes the "EMERGENCY EXIT" by itself.

The Frontend Timer (The Visual Guide):
When the frontend receives the alert, it pops up the warning box and starts a visual 30-second countdown for you to see. If you let it hit zero, it sends the "EMERGENCY EXIT" command to the backend (just in case the backend timer is a millisecond behind).
