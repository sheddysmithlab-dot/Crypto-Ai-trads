THE 5 PILLARS OF AUTOMATION (Kaun Kya Karta Hai?)
1. FRONTEND (Aapka Control Room)

Kaam: Yeh aapka Web App ya Mobile UI hai (jo design humne pehle dekha). Yeh sirf data dikhane aur aapke commands (START / STOP) lene ka kaam karta hai.

Wiring: Yeh directly Bybit se baat nahi karta. Yeh sirf aapke Backend se juda hota hai.

2. BACKEND (Factory ka Main Engine)

Kaam: Yeh ek server hai (jaise Node.js ya Python par bana hua) jo hamesha 24/7 chalta rahta hai. Yeh Frontend aur Bybit ke beech ka 'Middleman' hai.

Wiring: Frontend se commands leta hai, AI Agent ko jagata hai, aur Bybit API ko sambhalta hai.

3. AI AGENT (Factory ka Smart Manager)

Kaam: Yeh Backend ke andar baitha hua 'Dimag' (Algorithm) hai. Iske andar aapki saari policies feed hain (jaise 0.07+ par lock lagana, 0.02 par sell karna, aur 2.5% loss par emergency exit).

Wiring: Yeh lagatar live market data ko monitor karta hai aur decide karta hai ki kab 'Sell' ka button dabana hai.

4. API (Data ki Cable / Bridge)

Kaam: API woh rasta hai jisse Backend aur Bybit aapas me data share karte hain.

Wiring (2 Tarah ki hoti hai):

WebSocket API: Yeh ek 'Live Pipe' hai. Ek baar connect hone ke baad, Bybit har millisecond me current price aur portfolio balance is pipe ke through AI Agent ko bhejta rehta hai.

REST API: Yeh 'Action Cable' hai. Jab trade karni hoti hai, tab AI Agent iske through Bybit ko "SELL NOW" ka order bhejta hai.

5. BYBIT (The Market / Execution Ground)

Kaam: Asli crypto exchange jahan aapka paisa aur trades hain. Yeh sirf wahi karta hai jo Backend usko API ke through order deta hai.

🔄 THE AUTOMATION LOOP (Pura Wiring Flow Kaise Kaam Karta Hai)
Aaiye dekhte hain ki jab aap START button dabate hain, toh exactly data kaise flow karta hai (Pura Cycle):

Step 1: The Command (Frontend ➔ Backend)
Aapne Web App par START button click kiya. Frontend ne turant Backend ko ek signal bheja: "Bot ko on karo."

Step 2: The Connection (Backend ➔ API ➔ Bybit)
Backend turant jagta hai aur Bybit se WebSocket connect kar leta hai. Ab Bybit se live price aur aapke total portfolio ki value lagatar AI Agent ke paas aane lagti hai.

Step 3: The Thinking (AI Agent Live Monitoring)
AI Agent continuously data check kar raha hai.

Woh sochta hai: "Kya price 0.07 gaya? Haan gaya... Okay, lock activate karo."

Woh sochta hai: "Kya price 0.09 gaya? Haan... Lock ko 0.09 par shift karo."

Woh sochta hai: "Kya portfolio me 2.5% ka loss hua? Nahi, sab theek hai."

Step 4: The Action (AI Agent ➔ Backend ➔ Bybit)
Achanak price 0.09 se girkar 0.07 par aata hai (0.02 ka difference). AI Agent turant Backend ko bolta hai "Condition Met! Sell Karo!" Backend bina ek second lagaye REST API ke through Bybit ko SELL order bhej deta hai.

Step 5: The Feedback (Bybit ➔ Backend ➔ Frontend)
Bybit trade execute karke Backend ko confirm karta hai ki "Trade ho gayi". Backend yeh data database me save karta hai aur Frontend (aapke UI) ko bhej deta hai, jisse aapki screen par trade apne aap 'Active' se 'Closed' ho jati hai aur live profit update ho jata hai.

⚡ EMERGENCY KILL SWITCH WIRING (Bypass System)
Jab portfolio me 2.5% ka loss dikhta hai, ya aap khud laal STOP TRADING button dabate hain, tab system Step 3 ko bypass kar deta hai. Backend seedhe "Kill Command" bhejta hai aur Bybit me saari open positions ko Market Price par ek sath close kar deta hai. Uske baad WebSocket connection tod diya jata hai taki aur trade na ho.



Render Par Wiring Kaise Hogi?
Render par aap apne poore project ko do alag-alag hisson me host kar sakte hain:

Frontend (Web App): Aapke UI ko Render par as a "Static Site" (Free tier) host kiya ja sakta hai. Yeh bahut fast load hoga.

Backend & AI Agent: Isko Render par as a "Web Service" ya "Background Worker" deploy kiya jayega. Yeh continuously background me Bybit API se juda rahega aur data monitor karega.
